# Usage Scenarios
{: .no_toc }

This section describes how each IT profile uses ITIP in practice. Each sub-page covers a specific role's workflows, organized around the governance frameworks and DNA grammar introduced in [General Usage]({% link itip/general-usage.md %}).

Every profile interacts with the same governed definitions — the difference is *which* definitions they author, *which* they consult, and *which* views they use.

## Profiles covered

| Profile | Primary activity in ITIP |
|---------|------------------------|
| [Enterprise / Solution Architect]({% link itip/usage-scenarios/architect.md %}) | Define Structures, author Directives, compose framework stacks, produce architecture deliverables |
| [Developer / Tech Lead]({% link itip/usage-scenarios/developer.md %}) | Consult specifications, verify code-sourced definitions, check Norm compliance |
| [Project / Product Manager]({% link itip/usage-scenarios/project-manager.md %}) | Track governance readiness, monitor definition lifecycle state, coordinate deliverables |
| [Operations / SRE Engineer]({% link itip/usage-scenarios/operations.md %}) | Monitor compliance drift, inspect real-time architecture state, trigger remediation |
| [Security Officer]({% link itip/usage-scenarios/security-officer.md %}) | Audit regulatory coverage, verify security governance posture, manage security frameworks |
| [Compliance Analyst]({% link itip/usage-scenarios/compliance-analyst.md %}) | Evaluate regulatory adherence, generate compliance reports, track gap resolution |
| [QA / Test Engineer]({% link itip/usage-scenarios/qa-engineer.md %}) | Reference quality requirements as test criteria, verify quality governance coverage |
